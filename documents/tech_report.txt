1. Executive Summary

This report presents the complete technical architecture, system design, security framework, and infrastructure plan for the Coastal Innovation Summit website. The platform handles attendee registrations, speaker applications, stall bookings, partner and investor pages, and all public-facing event content.
 
The architecture is built on the MERN stack (MongoDB, Express.js, React, Node.js) and is engineered for zero monthly operating cost using permanent free-tier cloud services. The total cash budget is ₹700 — used entirely for transactional email via Brevo. All hosting, CDN, database, CI/CD, and monitoring services run on free tiers at no cost.
 
Security is implemented in ten independent layers covering DDoS, MITM, XSS, NoSQL injection, brute force, JWT theft, fake payments, CSRF, secret leakage, and data loss. Performance targets are Lighthouse score 95+, LCP under 1.5 seconds, and AVIF image delivery via Cloudflare.

5.2  Middleware Stack — Execution Order
Every request passes through this chain in order before reaching any route handler:
Helmet.js — attaches all security HTTP headers to every response
CORS — validates Origin header against the strict allowlist
express.json() — parses body with 10KB size limit (prevents large payload attacks)
Rate limiter — per-IP, per-route limits (see Section 7.4)
JWT verify middleware — on all protected routes only
Zod schema validation — on all POST / PUT routes
Route handler — pure business logic, no security logic here
Global error handler — catches all unhandled errors, logs to Sentry, returns safe JSON

8. Registration & Payment Flow

8.1  Attendee Registration — End-to-End Steps
User completes multi-step form on /register (React Hook Form, client-side Zod validation)
On final step, frontend sends POST /register/attendee with validated form data
Server middleware chain runs: CORS → rate limit → Zod schema validation
Controller calls Razorpay SDK: createOrder(amount, currency, receipt)
Server returns { orderId, amount, currency } to frontend
Frontend opens Razorpay checkout modal with the orderId
User completes payment (card / UPI / wallet / netbanking)
Razorpay sends webhook POST to /payment/verify on the server
Server verifies HMAC signature using crypto.timingSafeEqual — rejects if invalid
On verified success: (a) save registration to MongoDB, (b) append row to Google Sheet, (c) generate QR code, (d) send confirmation email via Brevo with QR attachment
Frontend receives 200 response, redirects user to /register/success with ticket summary
